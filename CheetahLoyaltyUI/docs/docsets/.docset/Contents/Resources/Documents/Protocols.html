  • AnalyticsManager

    This represents a class that manages analytics logs.

    Objects conforming this protocol must call observeAnalyticsLogs. By default, the observeAnalyticsLogs sets up the conforming object to observe notifications with names of Notification.Name.logEvent and Notification.Name.logScreen. The selector for the Notification.Name.logEvent notification is the logEvent(from:) method, while the selector for the Notification.Name.logScreen is logScreen(from:) method.

    This protocol is best to be conformed by the AppDelegate of an app. And then implementing the logEvent(from:) and logScreen(from:) methods with calls to the designated analytics service.

  • NotificationHelper

    Represents a type that setups and handles received local and remote notifications.

    Some of the methods of this protocol have default implementations throught protocol implementations. Please refer to the documentation of the methods.

    To setup the conforming type to receive and handle local and remote notifications:

    • Call registerForRemoteNotifications inside the application’s didFinishLaunchingWithOptions method.
    • Call setAsUserNotificationCenterDelegate to get UNUserNotificationCenterDelegate callbacks
    • Call handleNotification inside userNotificationCenter(_: UNUserNotificationCenter, didReceive: UNNotificationResponse, withCompletionHandler: @escaping () -> Void)
