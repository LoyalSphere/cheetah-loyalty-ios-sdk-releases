BaseTableController

open class BaseTableController<T> : NSObject, TableController

The base class for providing data and handling actions for a TableViewController.

This is a generic NSObject conforming to UITableViewDataSource, UITableViewDelegate, UITableViewDataSourcePrefetching, UISearchBarDelegate and ContentLoader protocols.

The associated type of this class will be used to initialize it’s DataProvider property. By default, the init(tableView:) initialization method assigns the table view property and sets it up using the setupTableView method.

This class manages paginated lists by keeping track of the current and loaded pages as well as prefetching succeeding pages. This shows a loading cell at the end of the list when it has not yet reached the last page.

By default, this uses TableRowCell for its cells. The associated type can conform to the TableRowCellPresentable to be able to configure the cells to present content of the type.

This class also supports data filtering through UISearchBarDelegate searchBar(_:,textDidChange:) method. It calls the updateFilteredData to update the filtered DataProvider property. The filteredData(with:) is called to filter the data using the search string.

When subclassed, the registerComponents and tableView(_:cellForRowAt:) can be overriden to bypass usage of the TableRowCell. The typical usage of this class when subclassed is:

  • Optionally override registerComponents to register cells and reusable views.
  • Override loadContent(completion:) to fetch and update content.
  • Optionally override tableView(_:cellForRowAt:) for configuring cells.

  • Override tableView(_:didSelectRowAt:) for cell select actions.

Note

Due to what maybe seems to be a bug with the Swift, methods not initially in the superclass and implemented in the subclass may not be executed. To get around this, the method needs to have an @objc attribue.

Update Methods

  • updateCollection(with:)

    Updates the collection view and collection data provider with the given items.

    This updates the collectionview and collection data provider with the following steps:

    • If newItems is empty, do nothing.
    • If currentPage is 0, remove all existing items. Then add new items, increment page and reload the collection view.
    • If currentPage is more than 0, perform batch updates by adding new items and cells.

    Swift
