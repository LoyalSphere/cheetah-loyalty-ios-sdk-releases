NotificationHelper

public protocol NotificationHelper : MessageNavigator, UNUserNotificationCenterDelegate

Represents a type that setups and handles received local and remote notifications.

Some of the methods of this protocol have default implementations throught protocol implementations. Please refer to the documentation of the methods.

To setup the conforming type to receive and handle local and remote notifications:

  • Call registerForRemoteNotifications inside the application’s didFinishLaunchingWithOptions method.
  • Call setAsUserNotificationCenterDelegate to get UNUserNotificationCenterDelegate callbacks
  • Call handleNotification inside userNotificationCenter(_: UNUserNotificationCenter, didReceive: UNNotificationResponse, withCompletionHandler: @escaping () -> Void)
  • handleNotification(request:with:) Default implementation

    Handle the specified notification request with date.

    The default implementation through protocol extension of this method first tracks the message id by calling the trackMessage(id:) method and then tries to retrieve the notification type from the notification payload and calls a method from the MessageNavigator protocol to handle the notification type. If no type has been retrieved or the type is unknown it calls the showMessages() of the MessageNavigator.

  • requestAuthorization(options:completion:) Default implementation

    Requests authorization to interact with the user when local and remote notifications are delivered to the user’s device.

    The default implementation through protocol extension of this method requests permission to display badge, alerts and play sounds. This calls the requestAuthorization(options:, completionHandler:) method of the UNUserNotificationCenter.

    Register to receive remote notifications via Apple Push Notification service.

This calls the registerForRemoteNotifications method of the UIApplication. This is usually called inside the application’s didFinishLaunchingWithOptions method.
