USER (U) : Creation & Authentication

  • create(newUser:completion:)

    Creates an Olo user. These are user accounts that have no associations with other login providers – i.e. the Olo user is the source of truth.

    With a successful response you will get a user authentication token which will be used to identify this user within Olo’s Ordering API going forward. This authtoken may be stored by the application but the user’s password may not.

    Important

    All reasonable security measures must be taken to secure the authtoken since it can be linked to the customer’s credit card.

    Declaration

    Swift

    public static func create(newUser: UserParameter.NewUser, completion: ResponseCompletion<User.New>)

    completion

    The closure to be executed once the request has finished
  • getOrCreate(userSSO:completion:)

    Links a user from an authorized (OAuth) third party login provider to a new Olo user. (Can also retrieve the details for an existing user previously created via this endpoint.)

    The source of truth for these users comes from the login provider, not Olo. Please provide a “provider” slug (can be found by contacting your Olo Customer Success representative) and the “providertoken” (to be retrieved from the third party login provider).

    With a successful response you will get an Olo user authentication token which will be used to identify this user within Olo’s Ordering API going forward.

    Important

    All reasonable security measures must be taken to secure the authtoken since it can be linked to the customer’s credit card.

    Note

    This will return an HTTP 204 if the login provider you provided is not yet authorized for the brand associated with your API credentials (in the respective Olo environment). If this happens, please reach out to your Olo Customer Success representative for more information.

  • deleteAccount(completion:)

    Deletes the user account associated with the provided current Olo authentication token.

    Note

    This action cannot be reversed and the user will need to create a new account if they wish to login in the future. All previously issued authentication tokens for the user will no longer be usable. Calling this endpoint using an authentication token obtained for an SSO linked user will only delete the user account in Olo’s system

    Declaration
